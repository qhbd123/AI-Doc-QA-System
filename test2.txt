AI智能文档问答系统 - 核心机制详解

1. 文档处理流程
- 上传：支持 PDF、DOCX、TXT 格式，单文件最大 10MB。
- 解析：根据文件类型调用对应解析器（PyPDF2、python-docx 或内置文本读取）。
- 分块：将长文本按 500 字符左右切分为语义完整的片段（chunk），保留段落边界。
- 存储：每个片段附带文档 ID 和时间戳，存入内存列表 `knowledge_base`，同时维护文档元信息 `documents_store`。

2. 检索算法（关键词匹配）
- 步骤1：将用户问题分词（使用 jieba 库）。
- 步骤2：遍历所有被选中的文档片段，计算每个片段与问题关键词的交集数量作为匹配分数。
- 步骤3：按分数降序排序，取前 3 个最高分片段。
- 步骤4：将选中片段按“【文档：xxx】\n片段内容”格式拼接为上下文。
- 若无任何匹配，则返回“无相关参考资料”，此时大模型将根据自身知识回答。

3. 缓存策略
- 缓存键格式：`{大模型提供商}:{问题小写}:docs_{文档标识}`
- 文档标识：若用户指定了文档 ID 列表，则使用排序后的 ID 用下划线连接；若未指定（检索全部文档），标识为 "all"。
- 缓存过期时间：默认 3600 秒（1 小时），可在 `.env` 中配置 `CACHE_EXPIRE`。
- 缓存清除时机：任何文档上传、删除或清空操作后，调用 `clear_qa_cache()` 删除所有 `doc_qa:*` 键，保证答案基于最新文档集。

4. 问答生成
- 构建提示词：若检索到文档片段，则使用“只根据下面的资料回答问题...”的 prompt；否则使用“请直接回答...”的 prompt。
- 调用大模型 API（豆包/DeepSeek/智谱），温度设为 0.1，保证答案稳定性。
- 返回答案同时附带 `source`（document/model）和 `sources`（引用文档名列表）。

5. 并发与性能
- 使用 `asyncio.Lock` 保护 `knowledge_base` 和 `documents_store` 的写操作。
- 缓存读写统计可通过 `/cache_stats` 接口查看命中率和平均延迟。
- 检索算法时间复杂度 O(N * M)，其中 N 为片段总数，M 为问题关键词数；1000 个片段内响应速度 <100ms。
